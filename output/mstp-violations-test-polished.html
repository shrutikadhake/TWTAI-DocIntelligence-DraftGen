User authentication API reference guide and documentation overview

Use the REST API to authenticate your requests. The system validates the token before granting you access. The server processes the request and returns a response.

Authentication methods and their descriptions

Select one of the following authentication methods before you can make API requests.

API key authentication method

After you complete registration, the system generates your API key and emails it to you. Store your API key in a secure location. Never share it with other users or third parties.

Include the key in the header like this:

Authorization: ApiKey YOUR_KEY_HERE

OAuth 2.0 authentication method and token management

Start the OAuth 2.0 flow by redirecting to the authorization endpoint. After you approve the access request, the system returns a code. Exchange the code for an access token and use it for subsequent requests.

Follow these steps:

  1. Open the authorization URL in a browser
  2. Enter your credentials
  3. Approve access
  4. Copy the code from the redirect URL
  5. Call the token endpoint with the code

Request parameters that need to be sent

The system requires the following parameters:

Parameter Type Description
client_id String The identifier the system uses to identify your application
redirect_uri String The URL the system uses to redirect you after authentication
scope String The permissions you request from the system
state String A random value you generate to prevent CSRF attacks

Error codes and their meanings

The system returns the following errors when it encounters problems during authentication:

Code Message Action
401 Token not valid Generate a new token
403 Resource forbidden Check your permissions
429 Too many requests Wait before retrying

What happens next

After you complete authentication, you can make API requests. Refresh your token every 24 hours by calling the refresh endpoint.
