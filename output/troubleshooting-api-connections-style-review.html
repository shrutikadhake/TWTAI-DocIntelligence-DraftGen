Summary

  • Total issues found: 19
  • Rules with violations: 4 of 5 — Active voice (9), Present tense (3), Sentence length (1), Heading case (6)

Requires human review

  • Error 401: Unauthorized and Error 403: Forbidden (heading case): "Unauthorized" and "Forbidden" are official HTTP status code names from RFC 9110. These are typically capitalised as proper terms in technical documentation. Confirm whether you want these lowercased for MSTP consistency or kept as official HTTP status labels.
  • Error 403, How to fix it, step 3 — future tense: "your IP address will need to be added by your admin" contains a genuine future event (the admin performing an action). The passive voice should still be fixed, but the future tense ("will need") may be intentional here since it refers to a real upcoming action, not system behaviour. Suggested fix: "Ask your admin to add your IP address" uses an imperative and avoids both issues.
  • H1 intro sentence — "This guide will help you": "will help" is technically future tense. This is a promise or description of the guide's purpose; MSTP would prefer "This guide helps you resolve…". Low priority but worth considering for full consistency.
