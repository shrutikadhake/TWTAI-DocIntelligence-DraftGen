# Dashboard User Manual: Reports Section

## Overview

The Reports section of the dashboard allows you to generate, view, and export data reports for your team. This section will help you understand how reports are created and what options are available to you.

---

## Accessing The Reports Section

To open the Reports section:

1. Log in to your dashboard at `app.company.com`
2. Click **Reports** in the left navigation menu
3. Select the type of report you want to generate

The Reports page will load within a few seconds. If it does not load, refresh the page or clear your browser cache.

---

## Types Of Reports Available

The dashboard currently supports three report types:

### Usage Report

A usage report will show you how many API calls were made during a selected time period. The data will be broken down by endpoint, status code, and date.

When to use it: When you need to monitor API consumption or identify endpoints that are being called most frequently by the users in your account.

### Error Report

An error report will be used by you to identify failed API calls and their causes. Each error will be listed with its status code, timestamp, and the endpoint that was called by the system.

> **Tip:** Error reports should be reviewed by you at least once a week to catch recurring issues early.

### Billing Report

A billing report will show the total API usage for the current billing cycle. The charges that will be applied to your account will be calculated automatically by the system based on your plan.

---

## Generating A Report

To generate a report, follow these steps:

1. Select the report type from the dropdown at the top of the page
2. Choose your date range using the calendar picker
3. Apply any filters you need (optional)
4. Click **Generate Report**

The report will be prepared by the system and will be displayed on screen within 30 seconds. For large date ranges, generation may take up to 2 minutes.

---

## Filtering Your Report

Filters allow you to narrow down your report data. The following filters are available:

| Filter | What it does |
|---|---|
| Endpoint | Shows data for one specific API endpoint only |
| Status code | Filters by HTTP status code (200, 400, 500, etc.) |
| Environment | Separates production and sandbox data |
| Team member | Shows calls made by a specific team member |

Multiple filters can be applied at the same time by you. To remove a filter, click the **X** next to it.

---

## Exporting A Report

After a report has been generated by the system, it can be exported by you in three formats:

- **CSV** — for use in spreadsheet tools like Excel or Google Sheets
- **PDF** — for sharing with stakeholders who do not have dashboard access
- **JSON** — for use in custom data pipelines or integrations

To export, click the **Export** button in the top right corner of the report and select your preferred format. The file will be downloaded automatically by your browser.

---

## Scheduling Automatic Reports

You can schedule reports to be generated and sent to your email automatically. This feature will be useful for you if you need regular updates without logging in manually.

To set up a scheduled report:

1. Generate the report you want to schedule
2. Click **Schedule This Report**
3. Choose the frequency — daily, weekly, or monthly
4. Enter the email addresses where the report should be sent
5. Click **Save Schedule**

The first scheduled report will be sent at the next occurrence of your chosen frequency.

---

## Troubleshooting Reports

### Report is taking too long to load

Large date ranges with heavy data will sometimes cause delays. Try reducing the date range or applying filters to narrow the data set.

### Export button is not visible

The export option will only appear after a report has been fully generated. Make sure the report status shows **Complete** before looking for the export button.

### Data seems incorrect

Report data will be refreshed every 15 minutes by the system. If you believe the data is wrong, wait 15 minutes and regenerate the report. If the problem persists, the support team should be contacted by you.
